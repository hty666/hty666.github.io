hexo是什么

如果一个初学者想用hexo搭建一个blog，首先要先了解一下hexo是什么

hexo实际上可以看到一个程序，是用来生成博客页面。在本地电脑上用hexo生成了博客页面，上传到github的仓库，就可以让大家访问仓库下的页面了，也就是访问博客，这就是Hexo的原理。

安装hexo

首先在计算机随便（你喜欢就好）一个盘里创建一个空的文件夹（hexo）,比如我的在F盘

右键这个文件，点击

打开github，新建一个项目，仓库的名字有一个专属的命名规则，比如（你github的用户名.github.io）,勾上Lnitialize this repository with a README，就创建好了，我这边已经有一个了所以会提示我此命名的库存已经存在了

建好之后进入这个库，点击右边的clone or download选择use SSH复制

进入你前面建好的blog文件夹，打开config.yml，翻到最底下，在type里输入git，type:(空格)git（在里面输入的时候在前面加个空格！！！不然不会生效），空格另起一行输入repo：（你在github里复制的网址，粘贴的时候要在前面加个空格！！！），空格再起一行输入branch： master（前面加空格再输入！！！）第 9 行的author改成你的名字

现在在git里面输入hexo g -d就是把你的hexo部署到git上面，成功之后，在网页上输入（你的用户名.github.io）就可以进去了（下面图片是我自己找的，因为我的主题已经设置过了会不太一样，所以用户名有问题不用怀疑）

如果想自己新建一个文件夹，打开blog里的source，进入_posts，右键创建一个.txt的文件，把后缀改成.md的文件就可以写啦，每次你更改里面的东西都要右键你的blog点击git bash here在里面输入hexo g -d，刷新一下服务器就可以了，这个东西传输的比较慢，如果不行耐心等待一会再刷新下界面试试

缺失模块。
1、请确保node版本大于6.2
2、在博客根目录（注意不是archer根目录）执行以下命令：
npm i hexo-generator-json-content --save
3、在根目录_config.yml里添加配置： 
jsonContent:
  meta: false
  pages: false
  posts:
    title: true
    date: true
    path: true
    text: false
    raw: false
    content: false
    slug: false
    updated: false
    comments: false
    link: false
    permalink: false
    excerpt: false
    categories: true
    tags: true
